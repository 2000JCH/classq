# 부하 테스트 결과

## 테스트 환경

- 도구: Gatling 3.11.5
- 대상 API: `POST /api/v1/enrollments` (수강신청)
- 테스트 강의: course_id=3 (UI/UX, 정원 30명)
- 가상 사용자: 100명 동시 투입 (atOnceUsers)
- 로컬 환경 (Docker Compose + Spring Boot)

## 시나리오

```
before(): 계정 생성 + MySQL/Redis 초기화 (측정 제외)
  ↓
100명 동시: 로그인 → 수강신청
  → 30명 성공 (201)
  → 70명 마감 (400 ENROLLMENT_CLOSED)
```

- HikariCP 커넥션 풀 튜닝

- 인덱스 추가 → 슬로우 쿼리 개선
