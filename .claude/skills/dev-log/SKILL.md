# dev-log

오늘 날짜에 맞는 Notion 개발 일지 페이지를 생성하거나 업데이트한다.

- 개발 일지 폴더: https://www.notion.so/3598d16f89fd80b5926fef8befd171cd
- 페이지가 없으면 오늘 날짜로 새로 생성, 있으면 기존 페이지에 새 내용만 추가

---

## 실행 절차

### 1단계 — 오늘 날짜 및 브랜치 확인

아래 명령어를 실행한다.

```
git log --oneline --after="어제날짜" --before="내일날짜"
git branch --show-current
git diff --staged --stat
```

- 오늘 날짜는 currentDate 컨텍스트에서 확인한다.
- 커밋 내역과 현재 브랜치를 파악한다.

### 2단계 — 스테이징 파일 확인 및 사용자 질문

`git diff --staged --stat` 결과에서 기존 대비 변경된 파일 목록을 확인한다.

변경된 파일이 있으면 사용자에게 아래 형식으로 질문한다.

> 커밋 내역을 확인했습니다.
> 스테이징 영역에 기존 대비 변경된 파일이 있습니다:
> - `파일명1`
> - `파일명2`
>
> 어떤 파일들을 기반으로 개발 일지를 작성할까요?

변경된 파일이 없으면 커밋 내역만 기반으로 작성한다고 안내하고 진행한다.

### 3단계 — 사용자가 선택한 파일 내용 확인

사용자가 선택한 파일을 `git diff --staged` 또는 Read 도구로 읽어 추가된 내용을 파악한다.

### 4단계 — Notion 페이지 존재 여부 확인

개발 일지 폴더 페이지를 fetch해서 오늘 날짜(`YYYY-MM-DD`) 제목의 자식 페이지가 있는지 확인한다.

```
notion-fetch: https://www.notion.so/3598d16f89fd80b5926fef8befd171cd
```

### 5단계 — 페이지 생성 또는 업데이트

**페이지가 없는 경우**: 아래 템플릿으로 새 페이지를 생성한다.

```
parent: page_id = 3598d16f89fd80b5926fef8befd171cd
title: YYYY-MM-DD
```

**페이지가 있는 경우**: 기존 페이지를 fetch해서 현재 내용을 읽는다.
- 이미 작성된 내용과 새로 추가할 내용을 비교한다.
- 중복되는 내용은 추가하지 않는다.
- 새로운 내용만 해당 섹션에 추가한다.

애매하거나 불확실한 내용이 있으면 작성 전에 사용자에게 질문한다.

## 주의사항

- 스테이징 파일이라도 `.claude/` 하위 파일은 사용자가 명시적으로 선택하지 않는 한 제외한다.
- 기존 페이지 업데이트 시 이미 있는 내용은 절대 중복 작성하지 않는다.
- 불확실한 내용은 추측해서 쓰지 않고 사용자에게 질문한다.
